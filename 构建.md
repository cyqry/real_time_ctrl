* 交叉编译

  1. 关闭360

  2. windows下载docker desktop的exe后，
  
     * 最好先执行```wsl --update和 wsl --shutdown```
  
     * 再执行
  
       ```
       "Docker Desktop Installer.exe" install --installation-dir="D:/OpenSourceDev/docker/docker" --windows-containers-default-data-root="D:/OpenSourceDev/docker/myContainer" --wsl-default-data-root="D:/OpenSourceDev/docker/myWsl" --hyper-v-default-data-root="D:/OpenSourceDev/docker/myHyperV"
       
       //docker的根目录放深一点，因为要将wsl目录放在与根目录同级，因为根目录有管理员权限限制，wsl访问wsl目录可能出现拒绝，所以我们将wsl目录放在与docker根目录同级，这样就可以了。如果这里docker启动失败，那么卸载重新安装，安装命令去掉--wsl-default-data-root="D:/OpenSourceDev/docker/myWsl"参数，安装后启动成功后，更改resource的路径。hyperV路径同理，container目录也最好同级
       ```
  
  3. 重启后启动docker desktop，记住修改设置的resource的路径
  
     -- 若导致虚拟机出现问题:[解决：VMware Workstation 与 Device/Credential Guard 不兼容_vmware workstation与device/credential_山顶夕景的博客-CSDN博客](https://blog.csdn.net/qq_35812205/article/details/121050789)
  
  4. cargo install cross       -- cross是一个基于docker的交叉编译工具，可以实现跨平台的交叉编译，基本与cargo语法一致，如下
  
     直接运行命令(若还没有这个target，运行前开个全局代理？好像也不用)，等待编译完成
  
     ```
     cross build  --release  -p ctrl_server   --target x86_64-unknown-linux-musl  //其中 -musl 后缀的规范编译出来的产物不依赖与宿主机的c库，所以更通用
     ```
